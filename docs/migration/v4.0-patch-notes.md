# Upgrading within Nodus v4.0.x

This document covers breaking changes and notable additions across the v4.0.x
patch series. Use it if you are already on v4.0.0 and upgrading to a later
4.0.x release.

If you are migrating from v3.x, start with [v3-to-v4.md](./v3-to-v4.md) instead.

---

## v4.0.1 — Breaking changes

### 1. `NodusRuntime()` sandbox defaults changed (security fix — #119, #97)

Two constructor defaults changed. Both are security fixes covered under §6
(security exception) of `COMPATIBILITY_MODEL.md`.

**`timeout_ms` — was 200 ms, now `None` (no deadline):**

```python
# v4.0.0 — 200ms deadline applied silently
runtime = NodusRuntime()

# v4.0.1+ — no deadline by default
runtime = NodusRuntime()   # equivalent to timeout_ms=None

# Restore 200ms explicit guardrail for short-lived sandboxed scripts:
runtime = NodusRuntime(timeout_ms=200)
```

If your v4.0.0 embedding relied on the 200ms default to kill runaway scripts,
add `timeout_ms=<N>` explicitly.

**`allowed_paths` — was unrestricted (`None`), now `[os.getcwd()]`:**

```python
# v4.0.0 — scripts could read/write any path the OS user could access
runtime = NodusRuntime()

# v4.0.1+ — scripts are jailed to the working directory by default
runtime = NodusRuntime()   # equivalent to allowed_paths=[os.getcwd()]

# Restore unrestricted access (trusted code only):
runtime = NodusRuntime(allowed_paths=None)

# Explicitly widen to a known safe path:
runtime = NodusRuntime(allowed_paths=["/app/data", "/tmp/scratch"])
```

If your v4.0.0 scripts read or write paths outside the working directory
without an explicit `allowed_paths`, they will now get a sandbox error. Pass
`allowed_paths=[<dirs>]` to re-allow those paths.

---

### 2. `httpx` is now an optional dependency

`pip install nodus-lang` no longer installs `httpx`. `std:http` builtins emit
a clear install-hint error when httpx is absent rather than crashing with an
import error.

**Restore http support:**
```bash
pip install "nodus-lang[http]"    # http only
pip install "nodus-lang[server]"  # http + FastAPI/Uvicorn
```

---

### 3. Integer division returns an integer

`6i / 2i` now returns `3` (integer floor division) instead of `3.0` (float).
Mixed-type division `6i / 2.0` still returns float.

```nd
// v4.0.0
print(6i / 2i)   // 3.0

// v4.0.1+
print(6i / 2i)   // 3
```

If your code passes integer division results to functions that expect floats,
or compares them with float literals, add an explicit conversion.

---

### 4. Division and modulo by zero now raises an error

`1.0 / 0.0` no longer returns `inf`; `0.0 / 0.0` no longer returns `nan`.
All division and modulo by zero (integer and float) now raise
`runtime_error("math", ...)`.

If you added `math.is_nan` / `math.is_inf` guards specifically for v4.0.0
float division, remove them — they are no longer reachable.

---

### 5. `NodusRuntime.last_vm` renamed to `_last_vm`

The `last_vm` attribute has been renamed to `_last_vm` to signal it is an
implementation detail.

**Migration:** Replace `rt.last_vm` with `rt._last_vm`, or switch to the new
stable API:

```python
stats = rt.get_execution_stats()
# {"instructions_executed": int, "coroutines_spawned": int}
```

## v4.0.1 — New additions

These are all opt-in and do not break existing code.

### `@annotation` syntax

```nd
@retry(max_attempts: 3i, backoff_ms: 100i)
fn fetch_data(url) {
    return http.get(url)
}

@exactly_once
fn charge_card(amount) {
    // idempotent — safe to retry
}
```

`@retry` requires `nodus-retry` installed. `@exactly_once` uses `std:effects`.

### Compound assignment operators

`+=`, `-=`, `*=`, `/=` now work on variables, index targets, and field targets:

```nd
let x = 10i
x += 5i      // x is 15i
lst[0] -= 1i
rec.count *= 2i
```

### Multiline expressions

Function calls, list literals, and map literals can now span lines:

```nd
let result = some_function(
    arg1,
    arg2,
    arg3,
)

let items = [
    "first",
    "second",
    "third",
]
```

### New `NodusRuntime` capability flags

Fine-grained opt-out flags for sandboxed embeddings:

```python
runtime = NodusRuntime(
    allow_subprocess=False,   # block std:subprocess entirely
    allow_network=False,      # block std:http entirely
    allow_env=False,          # block std:env entirely
    allowed_commands=["git", "python"],  # subprocess allowlist
    allowed_hosts=["api.example.com"],   # http allowlist
    coroutine_timeout_ms=5000,           # per-coroutine deadline
    event_sinks=[my_observer],           # capability audit log
)
```

### SCHED-001: cooperative sleep excluded from deadline budget

When `timeout_ms` is set, time spent in `sleep()` calls no longer counts
against the deadline. Only active VM instruction execution consumes the budget.
A coroutine sleeping 4×100ms with `timeout_ms=500` now completes cleanly.

## v4.0.2 — Bug fixes

| Issue | What was broken | Fixed behaviour |
|-------|----------------|-----------------|
| #207/#208 | `@exactly_once` idempotency not enforced; return value always `nil` | Correct enforcement and return value |
| #209 | `allowed_commands` allowlist not enforced in embedded mode | Blocked binaries now rejected |
| #210 | `@retry` function body ran 0 times when `nodus-retry` missing | Clear error emitted instead of silent skip |
| #212 | `event_sinks` wired but never called | Sinks now receive all events |
| #211 | `[1i, 2i,]` trailing comma was a syntax error | Now valid |
