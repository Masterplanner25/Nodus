# Nodus Projects

Nodus projects are rooted by a `nodus.toml` manifest.

## Manifest

Minimal example:

```toml
[package]
name = "example"
version = "0.1.0"

[dependencies]
json = "1.0.0"
```

The `[package]` table defines project metadata.

The `[dependencies]` table defines local path or version-based dependencies.

## Project Layout

Recommended layout:

```text
my-project/
  nodus.toml
  src/
    main.nd
  .nodus/
    modules/
```

`src/main.nd` is the default project entrypoint.

## Commands

Initialize a project:

```bash
nodus init
```

This creates:

- `nodus.toml`
- `src/main.nd`
- `.nodus/modules/`

Install declared dependencies:

```bash
nodus install
```

Add a dependency from the local registry:

```bash
nodus add json
```

Remove a dependency:

```bash
nodus remove json
```

Run a script directly:

```bash
nodus run src/main.nd
```

Run a project directory:

```bash
nodus run .
```

If `nodus run` is executed inside a project root with no script argument, it runs `src/main.nd`.

## Lockfile

Installed dependency state is recorded in `nodus.lock`.

This file is generated by tooling and should be kept under version control for reproducible installs.

## Resolution Order

When importing modules, Nodus checks in this order:

1. project modules
2. `.nodus/modules`
3. stdlib
